## git  
### git基础操作
`git push` 推送到远程仓库  
`git pull` 从远程仓库拉取  
### 基础操作附加命令
`git push --set-origin` 在本地创建分支以后使用，设置远程仓库新分支并连接到本地同名分支  
`git push --hard` 尽量避免使用该命令，该命令会强制使用本地仓库覆盖远程仓库，会导致远程的修改丢失  
### git分支操作 
- `git checkout`支持的功能过多，为了语义更清晰，减少冲突，其功能已经拆分到`git restore`和`git switch`实现  

`git switch -c <branch>` 切换并创建新的分支  
`git checkout <commit hash/tag>` 切换到某个提取某个commit（进入头部指针分离状态）  
`git restore <file>` 从当前头部指针从新获取某个文件  
`git rebase <branch>` 将当前分支变基到某分支，会导致git历史改变，保持修改历史线性（多人合作慎用,请该用git merge）  
`git merge <branch>` 将当前分支与某分支合并，类似各个仓库托管站pull request的效果
